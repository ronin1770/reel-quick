## 🚀 Getting Started

### 1. Fork the Repository

```bash
git clone https://github.com/your-username/reel-quick.git
cd reel-quick
```

### 2. Create a Branch

```bash
git checkout -b feature/your-feature-name
```

### 3. Make Changes

* Follow existing code style
* Keep changes focused and minimal
* Add comments where needed

### 4. Commit Properly

```bash
git commit -m "feat: add download button for completed videos"
```

---

## 📌 Contribution Guidelines

### ✅ Do

* Write clean, readable code
* Follow existing architecture
* Add logs where helpful
* Keep PRs small and focused
* Link issues in PRs

### ❌ Don’t

* Submit huge PRs without discussion
* Break existing functionality
* Add unnecessary dependencies
* Ignore linting / formatting

---

## 🧪 Testing

Before submitting a PR:

* Ensure your changes work locally
* Test edge cases
* Verify no regressions

---

## 📥 Pull Request Process

1. Ensure your branch is up to date
2. Open a Pull Request
3. Clearly describe:

   * What you did
   * Why it’s needed
   * Screenshots (if UI-related)

Example:

```markdown
## Summary
Added download button for completed videos

## Why
Users could not access generated files easily

## Changes
- Added download endpoint
- UI button added
```
